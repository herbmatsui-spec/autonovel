# 請求書自動生成システム — セットアップガイド

---

## システム全体の流れ

```
【事業所】 Googleフォームに入力・送信
    ↓（自動）
【フォーム回答シート】 データが自動蓄積
    ↓（総務がボタン一押し）
【総務入力シート】 利用料・請求日を確認・修正
    ↓（総務がボタン一押し）
【請求書シート（自動生成）】 利用者ごとに1シート
    ↓（総務がボタン一押し）
【PDF一括出力】→ Googleドライブに保存
```

---

## ファイル構成（Apps Script）

| ファイル名 | 役割 |
|---|---|
| `Code.gs` | メインロジック・メニュー定義・トリガー |
| `InvoiceBuilder.gs` | 請求書シートの生成（Excel書式を再現） |
| `PdfExporter.gs` | PDFエクスポート処理 |

---

## セットアップ手順

### ステップ1：スプレッドシートを準備する

1. Googleドライブで新しいスプレッドシートを作成
2. **拡張機能 > Apps Script** を開く
3. デフォルトの `コード.gs` を削除し、以下のファイルを追加：
   - `Code.gs`（＋マークから「スクリプト」を追加し、各ファイルの内容を貼り付け）
   - `InvoiceBuilder.gs`
   - `PdfExporter.gs`
4. **保存**して Apps Script エディタを閉じる
5. スプレッドシートをリロードすると「📄 請求書管理」メニューが追加される

### ステップ2：総務入力シートを作成する

1. スプレッドシートのメニューから **「📄 請求書管理」>「【初期設定】総務入力シートを作成」** を実行
2. `総務入力` シートが作成される
3. 利用者の情報を入力する：

| 列 | 内容 | 例 |
|---|---|---|
| 部屋番号 | 半角数字 | 207 |
| 利用者氏名 | フォームと同じ表記 | 中川　ツヤ子 |
| 請求年 | 西暦4桁 | 2026 |
| 請求月 | 数字 | 2 |
| 請求日 | 数字 | 10 |
| 利用料_単価 | 税別金額 | 49500 |
| 利用料_数量 | 通常1 | 1 |
| 管理料_単価 | 税別金額 | 10000 |
| 管理料_数量 | 通常1 | 1 |
| 共益費_単価 | 税別金額 | 20000 |
| 共益費_数量 | 通常1 | 1 |

> ⚠️ **部屋番号と利用者氏名はフォームの回答と完全一致させること**（紐付けキーになります）

### ステップ3：Googleフォームを作成する

1. Googleフォームを新規作成
2. **回答先のスプレッドシートをステップ1で作成したものに設定**
3. シート名を `フォーム回答_事業所` に変更
4. 以下の質問を追加（質問文は**完全一致**させること）：

#### 必須項目
| 質問文 | 種類 | 備考 |
|---|---|---|
| 事業所 | ラジオボタン | 第一事業所 / 第二事業所 / 第三事業所 |
| 部屋番号 | 短文 | 数字のみ |
| 利用者氏名 | 短文 | 総務入力シートと一致させる |
| 対象月（YYYY-MM） | 短文 | 例：2026-02 |

#### サービス料（最大10件、任意）
| 質問文 | 種類 |
|---|---|
| サービス1_内容 | 短文 |
| サービス1_数量 | 短文 |
| サービス1_単位 | 短文 |
| サービス1_単価 | 短文 |
| サービス2_内容 〜 サービス10_単価 | 同様 |

#### 消耗品（最大15件、任意）
| 質問文 | 種類 |
|---|---|
| 消耗品1_品名 | 短文 |
| 消耗品1_数量 | 短文 |
| 消耗品1_単位 | 短文 |
| 消耗品1_単価 | 短文 |
| 消耗品2_品名 〜 消耗品15_単価 | 同様 |

### ステップ4：フォーム送信トリガーを設定する（任意・推奨）

事業所がフォームを送信したら**自動で**請求書シートを更新したい場合：

1. Apps Script エディタを開く
2. 左の「⏰ トリガー」を選択
3. 「トリガーを追加」をクリック
4. 以下を設定：
   - 実行する関数：`onFormSubmit`
   - イベントのソース：スプレッドシートから
   - イベントの種類：フォーム送信時
5. 保存（権限の承認が求められる）

## 毎月の運用手順

### 事業所側（各事業所の担当者）
1. Googleフォームを開いてデータを入力・送信

### 総務側
1. **「① 事業所データを請求書に反映」** を実行  
   → 全事業所のフォーム回答から利用者ごとの請求書シートが自動生成される

2. 各利用者の `総務入力` シートの内容を確認・微調整  
   （利用料単価・請求月・請求日など）

3. **「① 事業所データを請求書に反映」** を再度実行  
   → 修正内容が請求書シートに反映される

4. **「② 全員分PDF一括生成」** を実行  
   → Googleドライブの「請求書PDF_YYYYMM」フォルダにPDFが一括保存される

## よくある問題

| 症状 | 原因と対処 |
|---|---|
| 請求書シートが作成されない | フォームの質問文と `Code.gs` の `get('...')` の文字列が不一致。フォームの質問文を確認 |
| 利用料が空欄になる | `総務入力` シートの部屋番号・氏名がフォーム回答と一致していない |
| PDFが空白 | シート名が `請求書_` で始まっていない可能性がある |
| 権限エラー | スクリプトを初回実行時に権限承認が必要。「許可を確認」から承認 |
