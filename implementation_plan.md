## 詳細実装手順

### 1. UI基盤コンポーネントの刷新
- Shadcn UI から `dialog` と `select` コンポーネントを `npx shadcn-ui@latest add dialog select` で追加。
- `src/components/ui/dialog.tsx` と `src/components/ui/select.tsx` を作成（shadcn が生成）。
- 既存のカスタムダイアログ（例: エラーダイアログ、確認ダイアログ）を新しい Dialog コンポーネントに置き換え。
- セレクトボックス（モデル選択、アーキタイプ選択など）を新しい Select コンポーネントに置き換え。
- スタイルは Tailwind とクラスバリアンスで調整。

### 2. 状態管理の構造化 (Zustand 導入)
- `npm install zustand`
- `src/store/useStore.ts` を作成し、以下のスライスを定義:
  - `user` (認証状態、ユーザー情報)
  - `uiState` (サイドバー開閉状態、テーマ、アクティブタブなど)
  - `books` (既存の useBooks ロジックを移行または併用)
- コンポーネントから `useBooks` を徐々に `useStore` の `books` スライスに置き換える。
- パフォーマンスのために、必要な部分のみを選択的にサブスクライブ。

### 3. ナビゲーション構造の再設計 (パンくずリスト)
- `src/components/ui/breadcrumb.tsx` を作成 (Shadcn UI の別コンポーネントまたは自作)。
- パンくずリストは現在のルート（例: ホーム > プロジェクト名 > 書籍名 > 編集）を表示。
- `App.tsx` のヘッダーまたはサイドバー上部に配置。
- ルート情報は `useLocation` (react-router-dom) また独自の状態から取得。

### 4. 非同期処理のUX改善 (その他のデータロードにスケルトン)
- 既存の `Skeleton` コンポーネントを `src/components/ui/skeleton.tsx` からインポート。
- プロット一覧、章一覧、キャラクター詳細、サマリー表示など、データ取得中のコンポーネントにスケルトンを適用。
- スケルトンの形状は実際のデータと同様のレイアウトを模倣（例: 章カードのサムネイル、タイトル、進捗バーなど）。

### 5. 通知システムの統合 (全ての alert を toast に置き換え)
- ソースコード内の `alert(` を全文検索。
- 各 alert を `import { toast } from 'sonner';` に置き換え、適切なメッセージとタイプ (success, error, warning) を設定。
- 例: `alert('保存に失敗しました')` → `toast.error('保存に失敗しました')`

### 6. 楽観的更新 (Optimistic UI) の拡張
- 書籍削除以外のミューテーション (プロット更新、章作成・更新・削除、キャラクター編集など) に適用。
- 各ミューテーション関数で:
  1. 楽観的に状態を更新 (例: 新しい章を即時にリストに追加)。
  2. API リクエストを送信。
  3. 成功ならそのまま、失敗ならロールバックし、エラートーストを表示。
- `useBooks` フックまたは `useStore` のミューテーションにラップ関数を作成して共通化。

### 7. インタラクティブなデータ可視化の強化 (ズーム/パン)
- `npm install react-chartjs-2-chartjs-zoom-plugin` (または同様のプラグイン)。
- `NarrativeGraph.tsx` でプラグインをチャートオプションに追加:
  ```js
  import { zoomPlugin } from 'react-chartjs-2-chartjs-zoom-plugin';
  options: {
    plugins: {
      zoom: {
        zoom: {
          wheel: { enabled: true },
          pinch: { enabled: true },
          mode: 'xy',
        }
      }
    }
  }
  ```
- ツールチップの改善は既に実施済みのため、ズーム/パンのみ追加。

### 8. 入力バリデーションのリアルタイム化 (Zod 導入)
- `npm install zod`
- 各フォームに対して Zod スキーマを定義:
  - イージーモード作成フォーム: アーキタイプ、トーン、長さなど。
  - 章インポートフォーム: ファイル、タイトル、内容など。
- `react-hook-form` と組み合わせて使用 (既に使っている場合は連携)。
- 入力変更ごとにバリデーションを実行し、エラーメッセージをリアルタイムで表示。
- 送信前に最終バリデーションを実行。

### 9. テーマ機能の実装 (ダーク/ライト切り替えスイッチ)
- `npm install next-themes`
- `_app.tsx` または `main.tsx` で `ThemeProvider` をラップ:
  ```tsx
  import { ThemeProvider } from 'next-themes';
  <ThemeProvider attribute="class" defaultTheme="system" enableSystem>
    <App />
  </ThemeProvider>
  ```
- サイドバーまたはヘッダーにテーマトグルボタンを追加:
  ```tsx
  import { useTheme } from 'next-themes';
  const { theme, setTheme } = useTheme();
  <button onClick={() => setTheme(theme === 'dark' ? 'light' : 'dark')}>
    {theme === 'dark' ? '☀️' : '🌙'}
  </button>
  ```
- Tailwind のダークモードクラス (`dark:`) をコンポーネント全体に適用済みであることを確認。

## リスクと対策
- **状態移行のバグ**: Zustand への移行時に状態が不整合になるリスク。対策: 移行は段階的に行い、各コンポーネントごとにテストを行う。
- **ライブラリの互換性**: Shadcn UI や zod のバージョンが既存の依存関係と競合する可能性。対策: 事前に `npm install --legacy-peer-deps` を使用し、必要に応じてバージョンを調整。
- **UIの一貫性**: 新コンポーネントのスタイルが既存とずれないように、Tailwind の設定とクラス名を統一。

## 完了の定義
- すべてのタスクが「完了」ステータスに移行し、ユーザーからのフィードバックで UI/UX が向上していることを確認。
- コードベースに重大なバグがなく、E2E テストまたは手動テストで主要フローが正常に動作すること。
