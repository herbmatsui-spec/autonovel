# 期待される挙動と現状の不十分な挙動の差分

## 現状の課題（不十分な挙動）
既存のテストコード（`tests/test_deai_auditor_edges.py`, `tests/test_critique_edge_rejection.py`）では、`SharpEdgeSpec` の `description` フィールドに対して、単純な文字列が含まれているか否か（`if description in content` 等）で判定を行っている可能性がある。

しかし、実際のLLM運用環境では、`description`（例: "余韻のある終わり方"）に対して、LLMは以下のような verbose（冗長）な表現を出力する。

- **LLM出力例**: "シーンの最後において、読者に深い余韻を残すような、静寂に包まれた幕引きが演出されています。"

現状のロジックでは、このverboseな出力から「余韻のある終わり方」というリテラルな文字列が抽出できない（またはマッチしない）場合、本来保持されている尖り（edge）を「欠落している」と誤判定し、False Positive（不要な拒絶）を引き起こしている。

## 期待される挙動
判定ロジックが「リテラル文字列の一致」から「意味的な要件の充足」へ移行すること。
具体的には：
1. 本文が冗長であっても、尖りのエッセンス（Semantics）が含まれていれば判定は True (PASS) になること。
2. 尖りが本当に欠落している場合（文脈的に意図が反映されていない場合）は、False (FAIL) になること。
3. 判定に用いる Description は、人間が読むための要約ではなく、LLMが判定するための「抽出可能な要件」であることを保証すること。
