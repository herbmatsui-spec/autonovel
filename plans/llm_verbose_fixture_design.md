# LLMVerboseFixture 設計仕様書

## 目的
実LLMが生成しがちな冗長な説明文（Verbose Description）をシミュレートし、尖り保全テストにおいて「リテラル一致に依存しない意味論的な検証」を可能にするためのフィクスチャを提供すること。

## 設計案

### 1. フィクスチャの配置
- ファイルパス: `tests/fixtures/verbose_edges.json` (データ)
- 管理クラス: `tests/fixtures/llm_verbose_fixture.py` (ロジック)

### 2. データ構造 (`verbose_edges.json`)
`edge_type` をキーとし、複数の冗長な説明文のリストを持つ構造とする。これにより、テスト実行時にランダムにサンプルを選択し、テストの堅牢性を高める。

```json
{
  "ending_pullback": [
    "シーンの終盤において、読者に深い余韻を残すような、静寂に包まれた幕引きを演出する。",
    "物語の結末に、あえて全てを語り尽くさない空白を作り、心地よい後味と余韻を演出する。",
    "静かな幕引きを行い、読者が自ら思考を巡らせるための精神的な余白を残して終了する。"
  ],
  "protagonist_flaw": [
    "主人公が決定的な選択を迫られた際、迷い、葛藤し、なかなか答えを出せないという優柔不断な性格を際立たせる。",
    "重要な局面において、自信のなさと迷いからくる優柔不断な態度が、物語の停滞や葛藤を生む要因となる。",
    "合理的判断ができず、感情的な揺れによって決断を先延ばしにする優柔不断な側面を描写する。"
  ],
  "sharp_conflict": [
    "二人の登場人物の間で、互いの譲れない価値観が激しく衝突し、張り詰めた緊張感が走る激しい議論を展開させる。",
    "単なる口論ではなく、信念と信念が真っ向からぶつかり合う、鋭く激しい対立構造を明確に描く。",
    "妥協を許さない二人の関係性から生じる、火花が散るような鋭い対立シーンを構築する。"
  ]
}
```

### 3. `LLMVerboseFixture` クラスの機能
- `load_samples()`: JSONファイルからサンプルデータをロードする。
- `get_random_description(edge_type: str) -> str`: 指定された `edge_type` に対応する冗長文をランダムに1つ返す。
- `get_verbose_spec(edge_type: str) -> SharpEdgeSpec`: 冗長な説明文を持つ `SharpEdgeSpec` インスタンスを生成して返す。

## 実装上の留意点
- **互換性**: 生成するオブジェクトは `src.models.sharp_edge.SharpEdgeSpec` と完全に互換性があること。
- **拡張性**: 新しい `edge_type` が追加された際に、JSONファイルに追記するだけで対応できるようにすること。
- **決定論的テスト**: 必要に応じて、ランダムではなくインデックス指定でサンプルを取得できるメソッド（`get_description_by_index`）も実装し、特定のケースを再現可能にする。
