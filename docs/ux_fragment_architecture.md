# Streamlit Fragment-based UX Architecture

## 1. 設計目的
本アーキテクチャは、Streamlit アプリケーションにおける「バックグラウンドジョブのポーリングによる全画面再描画（Full Rerun）」を排除し、ユーザー体験（UX）を向上させることを目的として設計されました。

### 解決した課題
- **フォーカス喪失**: ポーリングによる `st.rerun()` 呼び出しが、ユーザーが入力中の `st.text_area` 等のフォーカスを強制的に外していた問題。
- **レイアウトシフト**: コンテンツの更新に伴う要素のガタつき（レイアウトジャンプ）。
- **サーバー負荷**: 高頻度な全画面再描画によるフロントエンド・バックエンド双方への不要な負荷。

## 2. コアアーキテクチャ

### 2.1 状態管理の単一真実源 (SSOT): `UIStateStore`
`st.session_state` への直接的な書き込みを禁止し、`UIStateStore` を介して状態を管理します。
- **AppStateModel**: Pydantic モデルを用いて型安全に状態を定義。
- **RuntimeState**: 実行時の一次的な状態（アクティブジョブID、UIロック等）を管理。
- **イベント駆動**: `subscribe` / `_notify` メソッドにより、特定状態の変更を検知して UI を更新する仕組みを提供。

### 2.2 独立更新単位: `@st.fragment`
ページ全体を再描画せず、特定のコンポーネントのみを更新するために Streamlit の Fragment 機能を活用しています。
- **ポーリングの分離**: `progress_fragment(run_every=2.0)` により、1秒〜2秒間隔で進捗のみを更新。
- **関心の分離**: ログ表示 (`log_fragment`)、トークン統計 (`token_usage_fragment`) などを個別のフラグメントに分離し、相互の干渉を防止。

### 2.3 レイアウト安定化戦略: `st.empty()`
コンテンツの差し替えによるレイアウトシフトを防止するため、`st.empty()` コンテナを戦略的に配置しています。
- **コンテナ固定**: `main_content = st.empty()` の内部で `container(border=True)` を使用し、実行中ダッシュボード $\leftrightarrow$ 完了メッセージの切り替えを同一座標で行う。
- **CSS 補完**: `inject_focus_css()` により、`stProgress` や `stMetric` に `min-height` を設定し、中身が空の状態でも高さを維持。

## 3. 主要コンポーネントの詳細実装

### 3.1 `progress_fragment`
バックグラウンドジョブを監視する中核コンポーネントです。
- **UI処理ロック**: `UIStateStore.is_processing()` をチェックし、フォーム送信中などのクリティカルな操作中はポーリングを一時停止。
- **指数バックオフ**: 通信エラー発生時、`min(2**fail_count, 30)` 秒の間隔で再試行を行い、ネットワーク不安定時のフリーズを防止。
- **マルチジョブ対応**: `run_key` を導入し、複数のジョブ（例：「執筆」と「監査」）を個別のフラグメントで並行監視可能。

### 3.2 `log_fragment`
大量のログ出力を効率的に処理します。
- **差分レンダリング**: 全ログを再描画せず、`last_log_idx` を管理して新規追加分のみを追記表示。

## 4. 実装ガイドライン

### 4.1 新しい UI 要素を Fragment 化する判断基準
以下のいずれかに該当する場合、Fragment 化を検討してください。
- 高頻度（1秒〜数秒おき）に更新される必要がある。
- 更新時に他の入力フィールドのフォーカスを維持したい。
- 更新範囲が限定的であり、ページ全体の再描画が不要である。

### 4.2 `st.rerun()` の回避策
Fragment 内で状態を変更し、ページ全体の更新が必要な場合は以下のフローに従います。
1. `UIStateStore` を更新して状態を永続化する。
2. `UIStateStore._notify()` を通じて関連コンポーネントに通知する。
3. どうしても全画面更新が必要な場合のみ、ユーザー操作（ボタン押下等）に紐づけて `st.rerun()` を呼び出す。

### 4.3 メモリ管理とクリーンアップ
`st.session_state` に一時的な制御変数（例: `poll_fail_count_{run_key}`）を保存した場合は、必ず `UIStateStore.clear_active_job(run_key)` 等のクリーンアップ関数内で `del` してください。
