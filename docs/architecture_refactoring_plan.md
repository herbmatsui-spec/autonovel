# アーキテクチャリファクタリング計画書

本ドキュメントは、システム全体の持続可能な成長と保守性向上のためのロードマップを定義する。

## 目標
1. 開発効率の向上（学習コストの低減）
2. システムの堅牢性向上（テストによる安全網）
3. 保守性の向上（標準フレームワークへの移行）

## フェーズ1: 安定基盤の構築 (Safety & Observability)
- [ ] 1.1 ADR (Architecture Decision Record) の運用開始
- [ ] 1.2 トレースIDのUI連携によるエラー可視化
- [ ] 1.3 テストカバレッジの向上（AIモック化・Null Object活用）

## フェーズ2: 責務の明確化と簡素化 (Simplification)
- Wrapper層（streamlit_app の UI ラッパー）を削除し、直接 `src.infrastructure.api` へインポートを統合
- [ ] 2.1 UI層とバックエンド層の境界厳格化（Facadeパターン導入）
- [ ] 2.2 コンテキスト・データモデル（Pydantic）の責務分割と平坦化
- [ ] 2.3 重複した状態管理（EventBus/StateMachine）の統廃合

## フェーズ3: フレームワーク標準化と将来対応 (Modernization)
- [ ] 3.1 自前AI制御ループから標準フレームワーク（LangGraph等）への移行
- [ ] 3.2 非同期処理・トランザクション（Outbox）の外部ミドルウェア化
- [ ] 3.3 依存性注入（DI）フレームワークの導入

---

## 運用ルール
- 全ての大きな設計変更は ADR として記録する。
- リファクタリング時は必ずフェーズ1で構築したテストを通すこと。
