# ADR 0002: AIオーケストレーションフレームワークへの移行方針 (LangGraph)

## Context
現在のAI執筆パイプライン（`GenerationLoopManager`）は、複雑な手続き型ロジックで制御されており、リトライ制御や状態遷移の可視化が困難である。

## Decision
自前で実装している制御ループを廃止し、LangGraph を利用したグラフベースのオーケストレーションへ移行する。
1. 各工程（プロット生成、執筆、監査、修正）をノードとして定義する。
2. 条件分岐（リトライ判定、品質チェック）をグラフのエッジ（遷移）として表現する。

## Consequences
- フローの可視化が可能になり、デバッグが容易になる。
- 複雑な条件分岐やリトライ制御をフレームワーク標準の機能に委譲できる。
- コードベースがスッキリし、個別のノード（エージェント）の開発に集中できる。
