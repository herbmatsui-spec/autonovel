# ADR 0001: システム近代化のためのリファクタリング方針

## Context
システム内に複雑な依存関係や、重複する状態管理機構（EventBus, StateMachine）、UIとバックエンドの密結合が存在し、新規参画者や将来の保守におけるコストが増大している。

## Decision
以下の9つの課題に対して段階的なリファクタリングを実施する。
1. ADRとドキュメントの整備
2. **Wrapper層（streamlit_app UIラッパー）の削除と `src.infrastructure.api` への直接インポート統合**
3. 状態管理・通信機構（EventBus/StateMachine）の統廃合
3. UIとバックエンドの境界の厳格化（Facadeパターンの導入）
4. テストカバレッジの拡充
5. AI制御ループの標準フレームワーク化
6. データモデル（Pydantic）の分割と平坦化
7. トレースIDのUI連携
8. 非同期処理のミドルウェア委譲
9. DIの導入

優先順位として、「可観測性の向上（トレースID）」「テストによる安全網」「ドキュメント化」から開始する。

## Consequences
- コードの複雑さを減少させる。
- 障害切り分けコストを下げる。
- フレームワーク活用により、長期的なメンテナンスコストを削減する。
