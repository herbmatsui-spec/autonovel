# 覇権小説自動生成ツール v3.0 - 中立的技術評価レポート

## 1. 概要
本ツールは **Streamlit フロントエンド + FastAPI/非同期バックエンド + LangGraph ワークフロー + 複数 LLM プロバイダ対応** という構成で、Web小説（カクヨム/なろう等）の企画・プロット・執筆・監査・納品までを一気通貫で自動化する **「覇権小説自動生成エンジン」** である。

コードベース規模: **約 50+ Python モジュール、推定 15,000+ 行**（テスト・設定・プロンプトテンプレート含む）

---

## 2. アーキテクチャ評価

| 層 | 技術選定 | 評価 |
|---|---|---|
| **UI** | Streamlit + カスタム CSS/JS + `@st.fragment` 部分再描画 | ★★★★☆ 実用的だが状態管理が複雑化しやすい |
| **状態管理** | Pydantic `AppStateModel` + `UIStateStore` (Observer パターン) + JSON 永続化 | ★★★★★ 型安全・疎結合・デバッグ容易 |
| **DI コンテナ** | 手書き `AppContainer` (Object/Factory/Lazy/Singleton Provider) | ★★★★☆ 依存性逆転は徹底だが、標準ライブラリ不使用は保守リスク |
| **ワークフロー** | LangGraph (StateGraph + MemorySaver チェックポイント) | ★★★★★ 条件分岐・リトライ・人間介入に最適 |
| **LLM ゲートウェイ** | `GeminiApiClient` / `OpenAIApiClient` + `@with_llm_retry` (指数バックオフ・スキーマフォールバック) | ★★★★★ 耐障害性・プロバイダ抽象化ともに優秀 |
| **データ層** | SQLAlchemy 2.0 async + Alembic + Repository/UnitOfWork | ★★★★☆ 本番グレード。接続プール・WAL・PRAGMA チューニング済み |
| **ベクトル検索** | ChromaDB + BM25 ハイブリッド + プリフェッチキャッシュ | ★★★★☆ RAG 基盤として十分 |
| **設定管理** | Pydantic `GlobalConfigModel` (SSOT) + `ConfigValidator` + TOML/YAML | ★★★★★ 型安全・バリデーション・環境差分吸収が優秀 |

**総合アーキテクチャスコア: 4.6/5.0**  
→ **「エンタープライズ級の関心分離・テスタビリティ・拡張性を実現している」** と言える。

## 3. 機能・ドメインロジックの深度

### 3.1 二大ワークフロー
| ワークフロー | ノード構成 | 特徴 |
|---|---|---|
| **PlotLangGraph** (5ノード) | `align_context → generate_blueprint → audit_plot → expand_scenes → save_plot` | 条件付きリトライ (`should_retry_blueprint`) で品質ゲート |
| **WritingLangGraph** (7ノード) | `prepare → drafting → audit → (critic/healing) → dogfeed → finalize` | **Actor-Critic ループ** + 早期終了 (NCS≥90) + gen_ctx キャッシュ + チェックポイント復帰 |

### 3.2 品質保証パイプライン (7段階)
1. **整合性監査** (キーワード照合・閾値判定)
2. **因果律監査** (論理矛盾・伏線回収・状態違反)
3. **ステートバリデーション** (キャラクター状態・世界観ルール)
4. **Critic フィードバック** (Major/Critical のみ抽出)
5. **外科的ヒーリング** (失敗箇所のみピンポイント書き換え)
6. **Dogfeeding 自己評価** (読者視点シミュレーション・NCS スコアリング)
7. **遅延パッチ登録** (次話以降への伏線・修正差分を DB 永続化)

### 3.3 独自ドメイン機能
- **ストレス/カタルシス曲線** (累積テンション・QOL・代償スコアの自動計算・再計算)
- **アーキタイプ駆動企画** (17種のプリセット・チート強度/代償/システム補助率スライダー)
- **文体 DNA 解析・継承** (サンプル文からスタイル抽出・次話へ適用)
- **マルチモード対応** (かんたん全自動 / 上級者半自動 / パイプライン高速 / インポート研磨)

**ドメイン深度スコア: 4.8/5.0**  
→ **「単なる LLM ラッパーではなく、Web小説特有の構造的制約（伏線・因果・感情曲線・代償）をコード化した本格的ドメインモデル」**。

## 4. コード品質・保守性

| 指標 | 現状 | 評価 |
|---|---|---|
| **型ヒント** | Pydantic v2 + `mypy strict=true` 全域適用 | ★★★★★ |

## 6. リスク・懸念事項 (中立的指摘)

| # | リスク | 影響度 | 推奨対応 |
|---|---|---|---|
| 1 | **手書き DI コンテナ** | 中 | `dependency-injector` 移行 or インターフェース縮小で標準化 |
| 2 | **プロンプトテンプレート 100+ ファイル** | 中 | `PromptRegistry` キャッシュヒット率監視・ホットリロード検証 |
| 3 | **Gemini 依存度高** (フォールバック実装済みだが未検証) | 低 | vLLM/Ollama 実環境でのスモークテスト自動化 |
| 4 | **StateGraph チェックポイント肥大化** (長期連載) | 中 | `MemorySaver` → `SqliteSaver` 移行計画 |
| 5 | **Streamlit 固有の状態管理複雑性** (`st.fragment` + `UIStateStore` 二重管理) | 中 | 片側 (UIStateStore) へ完全統一 or Zustand 風単一方向フロー化 |
| 6 | **NCS スコアリング等の主観的指標** | 低 | 人間評価との相関分析・閾値校正パイプライン追加 |

### 結論
**「Web小説生成というニッチかつ複雑なドメインに対し、エンタープライズグレードのソフトウェア工学プラクティス (DI、LangGraph、Actor-Critic、Repository/UoW、Pydantic SSOT、非同期リトライ、ベクトル検索、構造化ログ) を過不足なく適用し、実用レベルで動作するシステムに仕上げている。」**

**推奨アクション (優先順)**
1. **DI コンテナ標準化** (`dependency-injector` 完全移行) → 依存グラフ可視化・循環検知自動化
2. **チェックポイント永続化** (`SqliteSaver` 移行) → 長期連載・クラッシュリカバリ対応
3. **プロンプトレジストリ観測性** (ヒット率・レンダリング時間メトリクス) → 本番チューニング
4. **フロントエンド状態単一化** (UIStateStore 完全統一) → バグ温床排除
5. **NCS/品質指標の校正パイプライン** (人間評価×自動スコア相関) → しきい値最適化
